CNKI 数字人文文献本体

CNKI Literature Ontology for Digital Humanities

https://liboyang88.github.io/cnki-ontology#

概述

本本体针对CNKI（中国知网）数字人文主题期刊论文的元数据描述需求设计。核心设计理念为复用优先——在 Dublin Core FOAF BIBO 等成熟词汇表的基础上，仅定义极少量的自定义元素，兼顾标准化互操作性与 CNKI 数据的特殊性。

本体可描述 100 篇数字人文期刊论文及其作者、期刊、学术机构、关键词等实体，支持 Turtle、RDF/XML、JSON-LD 三种序列化格式。 实例数据通过 Go + GORM 从 MySQL 数据库以编程式 D2RDF 方式生成。

类层次结构

本体定义了 5 个类。其中 Article 与 Journal 与 BIBO 等价，Author 与 AcademicOrganization 是 FOAF 的子集。

:Article
≡ bibo:AcademicArticle
核心类 — 一篇 CNKI 期刊论文
:Author
⊂ foaf:Person
论文作者（人的子集）
:AcademicOrganization
⊂ foaf:Organization
作者所属学术机构
:Journal
≡ bibo:Journal
论文发表期刊
:Keyword
独立类，无父类
论文关键词，独立定义以支持语义查询

属性设计

对象属性（复用 Dublin Core）

属性定义域 → 值域语义
dc:creator:Article → :Author论文创作者
dc:publisher:Article → :AcademicOrganization作者所属机构
dc:source:Article → :Journal来源期刊
dc:subject:Article → :Keyword主题关键词

数据属性

属性来源类型基数说明
dc:titleDCrdfs:Literal≥1论文题名
dcterms:abstractDCTermsrdfs:Literal—摘要
dcterms:issuedDCTermsxsd:date=1发表时间
dc:identifierDCrdfs:Literal≤1DOI（可选）
dcterms:referencesDCTermsrdfs:Literal≤1CNKI原文链接
:myId自定义xsd:integer=1内部序号
:srcDatabase自定义xsd:string=1CNKI来源库类型

命名空间与使用

命名空间前缀

# 在 Turtle 中使用
@prefix : <https://liboyang88.github.io/cnki-ontology#> .

# 实例 URI 模式
:article/1  a :Article .
:author/5   a :Author .
:journal/1  a :Journal .

实例示例

:article/1 a :Article ;
    :myId     "1"^^xsd:integer ;
    :srcDatabase "期刊"@zh ;
    dc:title   "融入与共生：ChatGPT在数字人文中的风险应对与前景展望"@zh ;
    dc:creator :author/1, :author/2 ;
    dc:subject :keyword/1, :keyword/2, :keyword/3 ;
    dc:source  :journal/1 ;
    dcterms:issued "2024-05-29"^^xsd:date .
